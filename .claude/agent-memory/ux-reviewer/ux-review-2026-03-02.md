# UX Review: Korean Church Bulletin - 2026-03-02

## Critical Issues
1. WorshipCommittee `grid-cols-2` fixed, no responsive override — at 320px each column is ~140px, names truncate or overflow
2. AutonomousGroups `grid-cols-4` fixed header and rows — 4 columns at 320px = ~72px each, Korean text will overflow/wrap badly
3. FastingPrayer `grid-cols-4` fixed — same problem as AutonomousGroups
4. Font size 0.65rem–0.7rem in WorshipOrder rows — renders at ~10-11px on mobile, illegible
5. No loading state for async `loadBulletin()` — blank content flash on load

## Major Issues
1. Top nav touch targets ~30px tall (py-1.5) — fails WCAG 2.5.5 (44px minimum)
2. `--gold` (#b8963e) used as text color on white/cream — contrast ratio ~2.7:1, fails WCAG AA
3. Schedule `w-14` time column with fixed width may overflow on small screens
4. No focus-visible styles in globals.css — keyboard navigation has no visible indicator
5. PrintButton missing aria-label (emoji + Korean text, but no accessible label fallback)

## Minor Issues
1. Section titles use letter-spacing 0.2em on Korean — can look over-spaced and slightly unnatural at small sizes
2. Footer contact info uses `text-xs` in a single long line — wraps awkwardly on narrow screens, no separator fallback
3. `mb-8` on BulletinHeader creates extra gap on mobile where vertical space is premium
4. Admin/Calendar nav links use emoji (⚙, 📅) without aria-hidden — screen readers will announce emoji names
5. Sticky header `px-6` on small screens leaves little room for button text + print button to coexist

## Enhancements
1. Add skeleton loader while bulletin data fetches
2. Consider collapsible sections for dense data (WorshipCommittee, StaffMembers) on mobile
3. Service type badge in BulletinHeader could be larger / more prominent
4. OfferingAccounts account numbers could use `font-mono` with copy-to-clipboard affordance
